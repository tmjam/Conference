Cross-origin resource sharing (CORS)

  • Why is this important?
    • Same Domain Policy issues with trying to shares web services or other web base resources
    • Aid in securely allowing web resources being shared by modifying headers of the HTTP request
  • Security Flaws:
    • HTML 5 and JavaScript attacks on sites (using HTML 5 attributes in combonation with JavaScript)
    • Cross Site Scripting (XSS)
    • For further information on the security: html5security.org

<script>
$(function() {
    $('#file-input').change(function(e) {
        var file = e.target.files[0],
            imageType = /image.*/;

        if (!file.type.match(imageType))
            return;

        var reader = new FileReader();
            reader.onload = fileOnload;
            reader.readAsDataURL(file);
    });

    function fileOnload(e) {
        var $img = $('<img>', { src: e.target.result });
        var canvas = $('#canvas')[0];
        var context = canvas.getContext('2d');
            context.clearRect(0, 0, canvas.width, canvas.height);
            context.beginPath();

        $img.load(function() {
            context.drawImage(this, 0, 0);
        });
    }
});
</script>

Block Scoping

Lexical declarations with let and const

  1. 'let' creates a block scope variable
  2. 'const' follows scemantics of 'let' and creates a read only constant declaration
  3. Function declarations are also supported within blocks with ES6

Web development practical standards

  • Class names should be more descriptive to the nature of the content
  • Recommended way to use hyphens in the class names
  • Double hyphens represent modifier class
  • Use namespace in the names
  • Avoid extra elements in the markup
  • Don't use tables for any layout
  • More information on Web components spec
